1Lab 35: DataLake trên AWS
- Giới thiệu về DataLake:
- Hiểu DataLake như một kho tập trung lưu trữ dữ liệu thô và đã xử lý để phục vụ phân tích và khai thác thông tin.
- Nắm các đặc tính chính: thu thập tất cả dữ liệu, truy cập đa người dùng, và mô hình truy cập linh hoạt (batch, interactive, real-time, search).
- Hiểu cách các dịch vụ AWS (Glue, Athena, QuickSight) tích hợp để xây dựng một giải pháp DataLake hoàn chỉnh.

- Amazon Glue (dịch vụ ETL):
- Tìm hiểu về Glue Crawlers để phát hiện schema tự động và tạo Glue Data Catalog.
- Hiểu cách Glue ETL job sinh script Python tùy chỉnh cho chuyển đổi và nạp dữ liệu.
- Thực hành tạo Crawler để quét dữ liệu trên S3 và xây dựng Data Catalog cho truy vấn.

- Amazon Athena (dịch vụ truy vấn tương tác):
- Athena cho phép truy vấn dữ liệu trên S3 bằng SQL tiêu chuẩn mà không cần pipeline ETL phức tạp.
- Cấu hình Athena để truy vấn dataset trên S3 và hiểu mô hình chi phí (trả theo truy vấn).
- Khám phá các định dạng dữ liệu Athena hỗ trợ (CSV, JSON, ORC, Avro, Parquet).

- Amazon QuickSight (dịch vụ trực quan hóa dữ liệu):
- Tìm hiểu cách QuickSight kết nối với nguồn dữ liệu và xây dựng dataset để trực quan hóa.
- Tạo biểu đồ, phân tích và dashboard để thể hiện các chỉ số kinh doanh chính.
- Hiểu cấu trúc QuickSight: Datasource → Dataset → Analysis → Visual → Dashboard.

- Các bước triển khai:
- Tạo IAM Role với quyền S3FullAccess và AWSGlueServiceRole.
- Chuẩn bị cấu trúc bucket S3 (/data, /ref_data) và tải file lên.
- Thiết lập Kinesis Delivery Stream để liên tục đưa dữ liệu vào S3.
- Triển khai stack CloudFormation để tự động hóa hạ tầng.
- Chạy AWS Glue Crawler để catalog dữ liệu S3 và xác thực bằng Amazon Athena.
- Khởi chạy SageMaker Notebook qua Glue Studio để thực hiện workflow chuyển đổi dữ liệu bổ sung.		29/09/202529/09/2025
2Lab40:
- Ôn lại AWS Glue như một dịch vụ ETL quản lý (Extract – Transform – Load) hỗ trợ chuẩn bị dữ liệu cho phân tích.
- Hiểu workflow:
1. Tải dữ liệu thô lên Amazon S3.
2. Dùng Glue Crawler để phát hiện schema và tạo database trong Glue Data Catalog.
3. Chuyển đổi dữ liệu sang định dạng tối ưu như Parquet.
4. Truy vấn dữ liệu đã chuyển đổi bằng Amazon Athena.
- Cấu hình AWS Glue và Athena cho cập nhật schema tự động và lên lịch chạy crawler.
- Thực thi các truy vấn SQL trong Athena để phân tích:
- Top 10 tài khoản và dịch vụ AWS có chi phí cao nhất.
- Phân tích chi tiết chi phí theo dịch vụ và tag (ví dụ: cost_center).
- Phân tách các kỳ tính phí và loại sử dụng.
- Học cách tối thiểu hóa chi phí truy vấn Athena bằng:
- Sử dụng file Parquet nén.
- Hạn chế kết quả truy vấn bằng LIMIT.
- Cấu trúc dữ liệu hiệu quả cho truy vấn phân vùng.
- Khám phá phân bổ chi phí và tagging cho theo dõi chi phí cấp doanh nghiệp.		30/09/202530/09/2025
3Lab 60: Amazon DynamoDB

- Tổng quan:
- Nghiên cứu Amazon DynamoDB — dịch vụ cơ sở dữ liệu NoSQL được quản lý hoàn toàn, cung cấp hiệu năng nhanh và khả năng mở rộng mượt mà.
- Hiểu cách DynamoDB tự động quản lý hạ tầng, scale và sao chép dữ liệu giữa các Availability Zone.
- Khám phá các tính năng như chế độ dung lượng on-demand và provisioned, mã hoá dữ liệu khi lưu trữ, point-in-time recovery, và tự động xoá item khi hết hạn.

- Thành phần chính:
- Table: Tập hợp logic của các item (tương tự bảng trong DB quan hệ).
- Item: Bản ghi riêng lẻ trong table (tương tự hàng).
- Attribute: Thuộc tính trong item (tương tự cột).

- Khóa chính:
- Tìm hiểu về Partition Key và Composite Primary Key (Partition + Sort Key) để xác định duy nhất item.
- Hiểu cách composite key cải thiện tính linh hoạt khi truy vấn.

- Secondary Indexes:
- Thực hành sử dụng Global Secondary Index (GSI) và Local Secondary Index (LSI) để tối ưu hiệu năng truy vấn.
- Nhận biết DynamoDB hỗ trợ tới 20 GSI và 5 LSI cho mỗi bảng.

- Read Consistency:
- So sánh Eventually Consistent Reads (nhanh hơn, có thể đọc dữ liệu cũ) và Strongly Consistent Reads (luôn đọc dữ liệu mới nhất, chậm hơn và tốn kém hơn).

- Chế độ dung lượng đọc/ghi:
- On-Demand Mode: Phù hợp cho workload không dự báo trước, tự động scale theo nhu cầu.
- Provisioned Mode: Thích hợp cho workload có thể dự đoán với lưu lượng ổn định hoặc có kế hoạch.

- Triển khai:
- Khám phá AWS Management Console và AWS CloudShell để thiết lập và quản lý DynamoDB.
- Học cách dùng AWS SDK (Boto3) để tương tác chương trình với bảng DynamoDB.		01/10/202501/10/2025

5Hoạt động: Tham dự sự kiện AWS về AI Development Lifecycle và giới thiệu Kiro

- Tham gia sự kiện do AWS tổ chức tập trung vào Vòng đời phát triển AI, bao quát các giai đoạn chính từ chuẩn bị dữ liệu, huấn luyện và đánh giá mô hình đến triển khai và giám sát liên tục.
- Tìm hiểu cách các dịch vụ AWS như Amazon SageMaker, Bedrock và CodeWhisperer hỗ trợ quy trình phát triển và tối ưu mô hình AI.
- Tham dự phần giới thiệu chi tiết về Kiro – một giải pháp AWS mới nhằm đơn giản hóa và hợp nhất quản lý workflow AI giữa các đội.
- Hiểu cách Kiro tích hợp với các công cụ AWS khác để quản lý dataset, phiên bản mô hình và tracking experiment hiệu quả hơn, nâng cao hợp tác và quản trị.
- Khám phá các case study thực tế cho thấy tổ chức tận dụng hạ tầng AWS để tăng tốc tự động hoá nhờ AI, giảm thời gian huấn luyện và cải thiện độ tin cậy mô hình trong môi trường production.

Kỹ năng đạt được:
- Nâng cao hiểu biết về vòng đời phát triển AI trong hệ sinh thái AWS.
- Học các ứng dụng thực tiễn của quản lý vòng đời mô hình bằng dịch vụ AWS và Kiro.
- Tăng kiến thức về observability mô hình, tracking experiment và best practice triển khai.
- Củng cố vốn từ kỹ thuật và kỹ năng giao tiếp liên quan đến AI, ML và Cloud Computing.		3/10/20253/10/2025
